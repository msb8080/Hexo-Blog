# 博客长期维护写作模板（Hexo）

## 使用方式

1. 执行 `npx hexo new post "文章标题"`。
2. 按 `scaffolds/post.md` 的结构补全。
3. 发布前更新 `updated` 和“迭代记录”。

## 适用场景

- 方案调研
- 技术选型
- 部署教程
- 问题复盘

## 必填规范

- `title`：具体明确，包含主题词。
- `date`：首次发布时间。
- `updated`：每次更新时改为当前时间。
- `tags`：3-6 个关键词。
- `categories`：1-2 个分类。
- `description`：120 字以内摘要。

## 推荐章节顺序

1. TL;DR（3 条内结论）
2. 背景
3. 目标
4. 方案对比（表格）
5. 最终决策
6. 实操步骤
7. 风险与回滚
8. 迭代记录

## 发布检查清单

- 标题是否可检索（含核心关键词）
- 外链是否可访问
- 命令是否可复制执行
- 移动端排版是否正常
- 代码块语言标识是否正确
- 迭代记录是否更新

## 版本管理建议

- 小修：直接更新文章并改 `updated`。
- 大改：在“迭代记录”注明变更范围。
- 方案替换：保留旧方案，标注“已过时”与替代方案。
