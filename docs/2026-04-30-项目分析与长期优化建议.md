# Hexo-Blog 项目分析与长期优化建议（归档）

日期：2026-04-30（更新：2026-05-01）
项目：Hexo-Blog
站点定位：AI Composer 技术猿（AI 后端、面试总结、投资理财与副业研究）

## 1. 总体结论

当前项目已从"搭建可用"进入"可维护迭代"阶段。
优势在于流程化与文档化非常完整；短板在于发布链路本机耦合、校验覆盖不足、内容赛道偏宽但代表作储备不足。

1. 发布流程标准化
   - 已形成 `check -> preview -> release` 固定入口（`bin/blog-flow.sh`）。
2. 模板体系完善
   - `scaffolds/post.md`、`scaffolds/draft.md` 已具备结构化写作框架。
3. 文档体系完整
   - 架构、流程、工具、写作模板、季度复盘均已沉淀在 `docs/`。
4. 品牌定位初步成型
   - "AI Composer 技术猿"方向一致，首页与导航有明确意图。
5. README 已建立 ✅
   - 根目录 README 已覆盖技术栈、快速开始、目录说明、文档索引。
6. 校验逻辑已部分增强 ✅
   - `blog-flow.sh` 的 `check_front_matter` 已支持收集 git 变更文件（`collect_front_matter_targets`），优于纯"最新文章"模式。

## 3. 主要问题（按优先级）

### P0（高优先级）

1. 发布链路依赖本机 SSH 绝对路径（`bin/release.sh:50`）
   - `GIT_SSH_COMMAND='ssh -i /Users/shuaibomin/.ssh/id_github ...'` 硬编码本机路径，换机迁移成本高，自动化部署受限。
   - 注：`blog-flow.sh` 的 `run_release()` 不走此逻辑，但缺少 Node 版本校验，两套脚本职责不统一。
2. 校验仍存在覆盖盲区 ✅ 部分修复
   - `blog-flow.sh:45-55` 已收集 git 变更文件，但 fallback 逻辑（`blog-flow.sh:73-79`）仅扫描 `source/_posts/*.md`，不包含 `_drafts`，多文并行/草稿转正式稿时可能漏检。
3. 导航与内容承接存在不一致
   - `_config.fluid.yml:11` 导航 `/about/` 链接指向不存在的页面（`source/` 目录下无 `about/`），是空链路。

### P1（中优先级）

4. 文档与配置出现轻微漂移
   - `docs/ARCHITECTURE.md:40` 写的是 `title = 闵帅博的个人博客`，但 `_config.yml:6` 实际值为 `AI Composer 技术猿`，需统一。
5. 赛道铺设偏宽
   - 当前内容体量下，四条主线并行会稀释输出密度与读者预期管理。
6. 两套发布脚本职责不清晰
   - `bin/release.sh` 含环境校验 + 部署，`bin/blog-flow.sh run_release()` 含 clean/build/deploy 但无环境校验。建议收敛为单一入口。

---

## 4. 策略建议（核心原则）

1. 先"稳系统"，再"扩栏目"
2. 先做"代表作"，再做"导航扩展"
3. 分类保持克制（3~4 个一级分类），细分交给 tags
4. 建立季度复盘数据化机制，避免只写"主观总结"

---

## 5. 分阶段行动（30/90 天）

### 30 天目标

- [x] 建立仓库根 README 作为唯一入口说明 ✅ 已完成
- [ ] 完成导航-页面一致性修复（修复 `/about/` 空链路）
- [ ] 统一文档口径（修复 ARCHITECTURE.md 中 title 漂移）
- [ ] 收敛发布脚本（release.sh vs blog-flow.sh 职责统一）
- [ ] 产出 3~5 篇可检索、可复用、可传播的硬内容
- [ ] 缩窄主线为 2 条（建议：AI后端 + 面试总结）

### 90 天目标

- [ ] 上线 CI 构建校验（至少 clean + build）
- [ ] 将发布能力去本机耦合（环境变量/Secrets 驱动）
- [ ] 校验覆盖全量（fallback 时包含 `_drafts`）
- [ ] 固化分类法与系列页结构
- [ ] 执行一次完整季度复盘并形成行动闭环

---

## 6. 写作运营建议（长期）

1. 文章类型固定为 4 类：路线图、实战复盘、方法框架、阶段更新。
2. 每篇至少提供 1 个"可执行产物"：命令清单/检查表/决策模板/风险清单。
3. 标题采用"场景+结论"结构，提升检索与点击一致性。
4. 节奏建议：主线月更 2 篇，副线月更 <=1 篇，季度复盘 1 篇。

---

## 7. 成功判定指标（建议）

- 每月发布：>=3 篇（其中主线 >=2）
- 站内互链覆盖：新文至少链接 2 篇旧文
- 首页首屏点击：主线入口可达率提升
- 季度复盘完成率：100%
- 构建成功率：CI 连续稳定通过
