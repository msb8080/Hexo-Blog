# 10 篇高优先级选题清单（Q2-Q3）

## 使用规则

- P0：必须优先发布，构建主线代表作
- P1：承接主线，形成系列深度
- P2：补充视角，控制频率

---

## P0（优先发布，建议前 4 周完成 3 篇）

1. AI 后端学习路线图：从 API 调用到可扩展服务
- 类型：路线图
- 读者收益：明确阶段与能力地图
- 必含产物：阶段目标表 + 每周行动清单

2. 一次真实超时故障复盘：重试、超时、熔断如何协同
- 类型：实战复盘
- 读者收益：拿到可落地的故障处理框架
- 必含产物：故障时间线 + 排查 checklist

3. RAG 工程最小闭环：索引、召回、重排的落地顺序
- 类型：方法框架 + 实操
- 读者收益：避免“概念懂、系统搭不起来”
- 必含产物：架构图 + 参数/风险对照表

4. 后端面试答题框架：把“会做”讲成“会答”
- 类型：方法框架
- 读者收益：提升表达稳定性与结构化输出
- 必含产物：通用答题模板 + 3 个示例题

---

## P1（形成系列深度，建议 2 个月内完成）

5. 慢查询优化实战：从 SQL 画像到索引策略
- 类型：实战复盘
- 必含产物：优化前后指标对比表

6. AI 服务成本控制：Token、缓存与限流的组合拳
- 类型：方案对比
- 必含产物：成本监控指标清单

7. 项目表达模板：背景-动作-结果-复盘（附真实案例）
- 类型：方法框架
- 必含产物：可复用讲稿模板

8. CI/CD 在个人项目中的最小实践：从手动发布到可验证流水线
- 类型：工程实践
- 必含产物：流水线步骤清单 + 回滚预案

---

## P2（副线内容，低频但稳定）

9. 技术人的复盘系统：如何做季度复盘不流于形式
- 类型：复盘方法
- 必含产物：复盘评分卡模板

10. 副业方向验证框架：从灵感清单到 MVP 决策
- 类型：研究分析
- 必含产物：方向评估矩阵 + 终止条件清单

---

## 发布节奏建议

- 第 1 月：1、2、4
- 第 2 月：3、5、6
- 第 3 月：7、8、9（10 作为弹性位）

## 质量门槛（每篇发布前）

1. TL;DR 不超过 3 条，且可执行
2. 至少 1 张对比表或清单
3. 至少 2 条站内互链
4. 至少 1 条“风险与回滚”
