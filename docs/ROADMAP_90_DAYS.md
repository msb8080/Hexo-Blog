# Hexo-Blog 90 天优化路线图（执行版）

## 目标定义（North Star）

在 90 天内，把博客从“可发布”升级为“可持续增长的内容系统”：

- 工程上：发布稳定、可迁移、可验证
- 内容上：主线清晰、代表作成型、读者路径顺畅

---

## 阶段一：Day 1-30（打基础，控范围）

### 目标

- 修复体验断点
- 提升内容供给质量
- 降低维护认知成本

### 关键动作

1. 导航一致性修复
   - 检查菜单链接与实际页面是否一一对应
   - 缺失页面立即补齐或暂时下线入口
2. 赛道聚焦
   - 主线锁定：AI后端 + 面试总结
   - 副线降频：投资理财/副业探索
3. 代表作冲刺（3~5 篇）
   - 每篇必须有“问题-方案-执行-复盘”
4. README 建立
   - 根目录单入口说明：启动、写作、发布、文档索引
5. 文档口径统一
   - 统一 Node、发布流程、分支策略表述

### 里程碑（M1）

- 首页无无效入口
- 主线至少 3 篇正式内容上线
- README 完成并可指导新机器快速启动

---

## 阶段二：Day 31-60（强流程，上自动化）

### 目标

- 把“靠人记忆”改成“靠流程保障”
- 把“本机可用”改成“仓库级可用”

### 关键动作

1. CI 构建校验
   - 自动执行 install/clean/build
   - PR 或 push 即触发
2. 发布去本机耦合方案设计
   - 将硬编码 SSH 路径改为环境驱动
3. 校验范围扩展
   - 从“最新文章”升级到“变更文章集”或“全量扫描”
4. 系列页设计
   - 主线各建立 1 个系列入口页（写给谁、先看哪篇、更新节奏）

### 里程碑（M2）

- CI 可稳定发现构建问题
- 校验规则覆盖多文场景
- 主线内容具备“入口页 + 连载页”结构

---

## 阶段三：Day 61-90（做闭环，看数据）

### 目标

- 形成“发布-反馈-复盘-迭代”的闭环

### 关键动作

1. 季度复盘实战
   - 使用现有模板完成首次真实季度复盘
2. 指标看板最小化
   - 记录发文数、主线占比、互链率、更新率、Top 文章
3. 内容复用机制
   - 把高质量文章拆分为：清单版/问答版/复盘版
4. 下季度计划制定
   - 保留有效动作，淘汰低收益动作

### 里程碑（M3）

- 完成一轮季度复盘并产出可执行下季度计划
- 主线内容形成持续更新惯性
- 工程发布路径可迁移、可追溯、可验证

---

## 风险与预案

1. 风险：选题过宽导致执行分散  
   预案：坚持“主线优先”周计划，副线仅补位。
2. 风险：自动化投入过重拖慢发文  
   预案：先做最小 CI（构建校验），部署自动化后置。
3. 风险：复盘流于形式  
   预案：每次复盘必须输出“3个保留动作 + 3个淘汰动作”。

---

## 每周节奏（建议）

- 周一：选题确认（1 主线 + 1 储备）
- 周三：首稿完成（结构先行）
- 周五：校验发布（check/preview/release）
- 周末：数据记录与下周调整
